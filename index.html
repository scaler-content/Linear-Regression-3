Concept
Visualizing the MSE Surface in 3D
⏱ 1.5 min

When we have two parameters — w (slope) and b (intercept) — the MSE cost function forms a 3D bowl-shaped surface:

  • The X and Y axes represent values of w and b
  • The Z axis represents the corresponding MSE value
  • The bottom of the bowl is the minimum MSE — the point where our best-fit line lives

Gradient Descent works by starting at a random point on this surface and taking small steps downhill until it reaches the bottom. As it descends, you can watch the regression line simultaneously move toward the best fit.

💡 Intuition
The smoother and more bowl-shaped the MSE surface, the more reliably Gradient Descent finds the global minimum. For Linear Regression, this surface is always convex — so GD always converges.

Importance of Train-Test Split
⏱ 4 min

In Lecture 2, we evaluated our model on the same data we trained it on. That's unreliable — the model has already "seen" those points. We need a held-out test set.

The Split

Partition the data randomly into two non-overlapping sets:

🎓 Training Set (80%)
🧪 Test Set (20%)

What Split Ratio Should You Use?

Data SizeRecommended SplitReasoning
Small (< 5,000 rows)80 / 20Model needs more data to learn reliably
Large (10,000+ rows)70 / 30Larger test set gives more confident evaluation
⚠️ The Split Must Be Random
If your data is sorted (e.g., by year), taking the first 80% would mean the model never sees recent data during training. Always shuffle before splitting.

The Correct Workflow

1

Fit on X_train, y_train

The model learns parameters exclusively from training data

2

Predict on X_test

Run the trained model on data it has never seen

3

Evaluate on y_test

Compare predictions to true values — this R² is the honest measure

Scikit-Learn provides train_test_split to handle all of this automatically:

from sklearn.model_selection import train_test_split

X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.30, random_state=1
)

Interpretation
Feature Importance and Coefficient Interpretation
⏱ 2 min

Suppose we're predicting product sales from two ad channels:

ŷ = 3 + 2×TV + 3×SocialMedia

How to read this:

  • Holding TV budget fixed, a +1 unit increase in Social Media spend → +3 units of sales
  • Holding Social Media fixed, a +1 unit increase in TV spend → +2 units of sales
  • Social Media has a stronger positive effect on sales in this model

Negative Coefficients

ŷ = 3 + 2×TV − 3×SocialMedia

Now Social Media has a negative coefficient: more social media spending decreases predicted sales. This might reflect a poorly received campaign that triggered a boycott.

📌 Key Rule
Higher absolute value of a coefficient → stronger influence on the target. The sign tells you direction (positive = increases target, negative = decreases target). Comparing coefficient magnitudes is only valid when features are on the same scale — a reason to standardize.
⚖️
Preprocessing
Importance of Standardization
⏱ 2.5 min

Consider two features in our ad-spend model:

  • TV budget: values in the thousands (e.g., 50,000 – 500,000)
  • Social Media budget: values in single digits because it's expressed in millions (e.g., 1 – 30)

When the model learns coefficients, it compensates for scale:

  • It picks a tiny w₁ for TV (since TV values are huge, even a small w₁ has big impact)
  • It picks a large w₂ for Social Media (since SM values are small, w₂ must be large to matter)

This makes the raw coefficients incomparable and can also slow down Gradient Descent convergence.

Two Solutions

MethodFormulaResult
Z-score Standardization(xᵢ − μ) / σMean = 0, Std = 1
Min-Max Normalization(xᵢ − min) / (max − min)All values in [0, 1]

After scaling, coefficients are on comparable scales and can be used to directly rank feature importance.

📏
Preprocessing
Min-Max Scaler
⏱ 1.5 min

Min-Max scaling squeezes every feature value into [0, 1]:

x_scaled = (xᵢ − min) / (max − min)

Proof It Works

Input xᵢResult
xᵢ = min(min − min) / (max − min) = 0
xᵢ = max(max − min) / (max − min) = 1
Any value in betweenA value in (0, 1)

The Outlier Problem

⚠️ Critical Limitation
If a feature contains an extreme outlier, that outlier becomes the max. Every other data point then gets squashed into a very narrow band near 0 — e.g., [0.0, 0.18] — losing most of the original variance. Always handle outliers before Min-Max scaling.

Z-score standardization is less sensitive to outliers since it uses mean and standard deviation rather than the absolute min/max.

One-Hot Encoding
⏱ 5 min

The origin column contains categories: American, Europe, Asia. We can't feed strings into a regression model, so we need to convert them to numbers.

Why Not Just Use 1, 2, 3?

Assigning America=1, Europe=2, Asia=3 would imply an ordering that doesn't exist. The equation y = mx + c would treat Asia (3) as having three times the impact of America (1). That's mathematically wrong for a nominal category.

Dropping the Redundant Column

With 3 categories, knowing any 2 columns tells you the 3rd automatically (if not American and not European → must be Asian). One column is therefore redundant and can cause multicollinearity. We drop one:

Handling Missing Values
⏱ 5 min

The dataset uses ? to mark missing horsepower values — meaning the horsepower column was parsed as strings (object dtype) instead of numbers. Let's find and fix this.

Detect Non-Numeric Rows

hpIsDigit = pd.DataFrame(cData.horsepower.str.isdigit())
cData[hpIsDigit['horsepower'] == False]
Output — 6 rows with '?' in horsepower
mpg cylinders displacement horsepower weight ... 32 25.0 4 98.0 ? 2046 126 21.0 6 200.0 ? 2875 330 40.0 4 85.0 ? 1835 (+ 3 more rows)

Replace ? with NaN, then Convert Type

cData = cData.replace('?', np.nan)
cData['horsepower'] = cData['horsepower'].astype(float)

Impute with Median

Dropping 6 rows out of 398 is an option, but imputing with the median retains them without distorting the distribution (unlike mean, which is sensitive to outliers):

medianFiller = lambda x: x.fillna(x.median())
cData = cData.apply(medianFiller, axis=0)
Median horsepower used for imputation

📌 Why Median Over Mean?
If horsepower has skewed values or outliers, the mean would be pulled toward them. The median is more robust — it represents the "middle" value regardless of extremes.

AI-Powered Coefficient Narrative

Rather than manually writing a business interpretation every time, we can automate it using an LLM. The code below passes the feature names and coefficients to a Groq-hosted Llama model and asks it to explain the model in plain English for a non-technical audience:

import os

You are explaining a Linear Regression model predicting car mileage (mpg).

Coefficients:
{coef_text}

In plain English, explain:

2. The relative strength of each

Impact of Outliers on the Regression Line
⏱ 2 min

The regression line minimizes the sum of squared errors. Because errors are squared, outliers — which have very large errors — exert a disproportionate gravitational pull on the line.

  • Without outliers: the best-fit line cleanly follows the trend of the data
  • With an outlier: the line tilts toward the outlier to reduce its squared error, at the cost of fitting the rest of the data worse

The further an outlier sits from the cluster of normal points, the harder the line is pulled — like a rubber band stretched toward a pin.

📌 Best Practice
Always perform outlier detection and treatment before fitting a Linear Regression model. Common methods: IQR-based clipping, z-score filtering, or visual inspection via box plots.

Metric
Problems with R² — Adjusted R²
⏱ 1.5 min

The Problem with R²

R² has a fatal flaw: it always increases (or stays the same) when you add more features, even if those features are completely random noise. This means:

  • A model with 50 useless features will have a higher R² than one with 5 good ones
  • R² alone cannot tell you if a new feature is genuinely improving the model

Adjusted R² — The Fix

Adjusted R² penalizes the score for adding features that don't improve the model proportionally:

Adj. R² = 1 − (1 − R²) × (n − 1) / (n − p − 1)

where n = number of samples and p = number of features. Adding a useless feature slightly decreases Adjusted R².

y_hat = regression_model.predict(X_test)
n = len(y_test)
p = X_test.shape[1]
adj_r2 = 1 - (1 - regression_model.score(X_test, y_test)) * (n - 1) / (n - p - 1)
print("Adjusted R²:", adj_r2)
Output
Adjusted R²: 0.9040928655894155
📌 When to Use Which
Use R² for quick sanity checks. Use Adjusted R² when comparing models with different numbers of features — it's the fairer judge of actual model quality.

Introduction to StatsModels
⏱ 6 min

Scikit-Learn is optimized for prediction. StatsModels is optimized for statistical inference — it gives you the full picture of model significance, not just predictions.

Scikit-Learn vs StatsModels

Primary goalPrediction accuracyStatistical inference

Confidence intervals❌ No✅ Yes
Diagnostic tests❌ No✅ Yes

Fit with StatsModels OLS

import statsmodels.api as sm
from sklearn.preprocessing import MinMaxScaler

# Scale the data first
scaler = MinMaxScaler()
df_train_scaled = pd.DataFrame(scaler.fit_transform(df_train), columns=df.columns)
df_test_scaled  = pd.DataFrame(scaler.transform(df_test),  columns=df.columns)

y_train_sm = df_train_scaled['selling_price'].values
X_train_sm = df_train_scaled.drop('selling_price', axis=1)

# StatsModels does NOT add an intercept by default — add it manually
X_sm = sm.add_constant(X_train_sm)

model_sm = sm.OLS(y_train_sm, X_sm)
results = model_sm.fit()
print(results.summary())
